## 3. Convolutional Neural Networks.mp4

### Summary

Convolutional neural networks were specifically engineered to process images by overcoming the limitations of standard models, which often **destroy spatial relationships** by flattening pixel data into simple lists. Instead of assigning a unique parameter to every individual pixel, these networks utilize a **sliding window approach** where compact filters scan the image to detect patterns regardless of their location. This method ensures **parameter efficiency** and allows the model to aggregate local information into increasingly complex features through a growing **receptive field**. By preserving the natural structure of the pixel grid, these architectures have become essential tools for high-stakes visual analysis in fields like **medical imaging and radiology**.

**Keywords:** Convolutional Neural Networks, Spatial Information, Pixel Grid Format, Convolutional Filter, Receptive Field

### Content

Convolutional neural networks, otherwise known as CNN's, were designed with images in mind. Consider the pixel grid format that we discussed as a way of representing images. It's immediately clear that there is no natural way to represent this as a list of features as we did in the structured data scenario. Sure, we could flatten the pixel grids and string them together like this. But there are a couple of major issues with this. First, each parameter would be dedicated to only one or a handful of pixel locations at a time. This pres presents us with a problem. In nearly all practical settings, samples will present objects in highly variable positions. If we were trying to solve a classification problem based on the presence of cancer in an image, the placement of pixels representing cancer cells and even the basic structures within the body would be highly variable from patient to patient. If we were to use a fully connected layer as our first layer, each parameter would be isolated to finding patterns in only one pixel at a time. Even if we could train a model with enough parameters to find patterns across the whole image, we would miss out on a very natural aspect of the pixel grid, which is that pixels near each other are highly related. Flattening the image destroys the spatial information that we have about the image and places adjacent pixels thousands of numbers apart from each other. So what we need is a layer that is one parameter efficient and two preserves spatial information that's present in the pixel grids. For this we use what is known as a convolutional layer. The key building block of a convolution layer is comprised of a convolutional filter or kernel which consists of a small group of parameters that are multiplied and summed. In other words, a linear combination with one small patch or window of the image at a time. The output of each linear combination passed through an activation function such as the sigmoid function we discussed earlier is placed relative to the location of the input patch in a new sometimes smaller number grid. Let's visualize this one more time in a different way. The convolutional filter in this example is the shadow which itself is a 3x3 grid of parameters. This 3x3 grid of parameters slides around in this case a single input grid corresponding to a grayscale image which here is a large blue square. The values of the parameters are multiplied with the values of the pixel grid and then summed and passed through an activation function. This produces a single value which is then placed in the grid of activations here shown in green. You can think of the filter as a feature detector. Since only one value is produced for every 3x3 patch in the image in a trained model, one can imagine a high number being computed if something of interest is found in the patch and a low number if not. A convolutional layer then consists of many filters, each with a different set of parameters. Each filter can be thought of as scanning the pixel grid and looking for different patterns or objects of interest one patch at a time. Indeed, just like fully connected neural networks, one can stack convolutional layers as well. The grid of activations produced by one convolutional layer can very well act as the image input to the next layer. Since each layer outputs a single number for each small patch of its input, layers towards the end of the neural network end up seeing information collected from larger and larger segments of the image. The amount of the image that a given filter has information for is known as a receptive field. Such an architecture mitigates the issues surrounding fully connected neural networks through the following. Instead of each parameter being locked into a single pixel location. The sliding window-based paradigm allows each filter, which in turn is just a group of parameters to move freely about the image, which allows parameters to train in a way that is useful across the whole image. Each filter aggregates the information at a patch level, meaning that the model can leverage from the spatial information of the image. Convolutional layers are what makes CNN CNN's or convolutional neural networks. CNN's are trained just like dense neural networks, albeit computing the gradient for gradient descent becomes a slightly more complex procedure. CNN's are immensely popular in subdomains of medicine such as radiology, dermatology, and pathology among others that deal with images.

Deep learning represents an evolution of traditional machine learning by organizing vast numbers of parameters into **hierarchical layers of neurons**. Each neuron acts as a miniature processing unit that performs a **linear combination** of inputs followed by a **nonlinear activation function**, such as the Sigmoid or ReLU, to extract increasingly complex patterns. Unlike simpler models that interact with features directly, deep neural networks feed the outputs of one layer into the next, creating a **layered function structure** capable of mapping intricate relationships within data. This architectural complexity is managed through an iterative training process involving **gradient descent** and **backpropagation**, which refine the model's coefficients to minimize error. Ultimately, the power of these networks lies in their ability to transform raw data into **higher-level features** through repeated cycles of computation and transformation.

**Keywords:** Neural network architecture, Deep learning models, Nonlinear activation functions, Gradient descent optimization, Layered function structures

## 11. The Transformer Architecture for Sequences.mp4

### Summary

The transformer architecture marks a significant evolution beyond recurrent neural networks by utilizing **self-attention layers** to process entire sequences simultaneously rather than one element at a time. This design overcomes the limitations of sequential processing, which often struggles with **long-range dependencies** and the technical inability to **parallelize computations** for greater speed. By calculating a **contextual relatedness signal**, the model determines how much weight to give surrounding words, allowing the meaning of a specific element to be informed by its neighbors regardless of their distance. Ultimately, this approach creates **output representations** that are more nuanced and efficient, making transformers the preferred choice for complex tasks like natural language processing and genomics.

**Keywords:** Transformer architecture, RNN disadvantages, Self-attention layer, Long-range dependencies, Sequential vs parallel

### Content

One final architecture you may have heard of is the transformer architecture. The transformer architecture is quickly replacing RNN's in sequence-based problems. Let's revisit the RNN. RNN's rely on reading the sequence one element at a time in order to both process and memorize bits and pieces of information. While there are clear advantages of doing this over dense layers that read the entire input all at once, there are also some disadvantages. RNN start to lose effectiveness if they have to deal with long range dependencies. In a given sentence, an RNN usually feeds information to itself one time step at a time. If it needs to remember something from the beginning of the sequence in order to produce the right output at the end of the sequence, it may have difficulty capturing that information in the context that it passes to itself. Furthermore, on the technical side, it is hard to parallelize the processing of an RNN. One of the most powerful features of modern computing systems is the ability to do many tasks at the same time. For a given element in a sequence, RNN's need the information from the past elements in order to output predictions and therefore their computations have to happen sequentially. For long sequences, this becomes a problem. The transformer architecture is built around a layer known as a self attention layer which allows for the processing of sequences to happen all at once while at the same time producing outputs that are aware of the rest of the sequence. The fundamental assumption here is that the meaning of sequences in other words the sentences is not constructed linearly. The meaning of a given word changes based on the many words surrounding ing it. Take the following example of two sentences. I broke my leg while running versus I was trailing in the first leg of the marathon. The meaning of the word leg is informed by words both to the left and to the right of the word itself. Crucially though, the amount the meaning of the word is affected by its neighboring words varies significantly depending on the neighbor itself. Take the first example. The word broke sends a strong signal that leg here describes the biological appendage as opposed to a segment of a race. The word My may send a medium signal as one does not typically own segments of races. The word running may send a weak signal since the word leg could either be the appendage or the segment of a race. In the second example, trailing sends a strong signal along with first and marathon. The concept of strength of contextual signal is exactly what self attention layers try to capture. Self- attention layers compute a contextual relatedness signal or weight between every element and the others in an input sequence. Then broadly speaking, the element at each position is transformed using a weighted sum of feature values from the elements in other positions based on the strength of the contextual relatedness. Self- attention layers can produce multiple weighted outputs to encode different types of context. While the actual mathematical operations behind self attention are a little more complicated than what we've just described, the parameter values that are used to perform the operations are learned to optimize final loss objectives. Just just like all the other models we've discussed so far. And the key concept you should take away is that self attention layers transform their input into output representations that incorporate context at each element position in a variable way. In other words, they look at the entire input sequence and then pay attention to context elements variably based on what element values actually are. Because self attention looks at the entire input sequence at once, it avoids the problem with RNN's of potentially having to remember context over long sequences. It is also therefore much more efficient than RNN's which have to sequentially process input elements one by one. Self attention layers do pose some additional challenge in keeping positional information about how far away input elements are from each other, but there have been workarounds proposed to address this. Transformer architectures stack many self- attention layers on top of each other. So far, they've proven themselves to be both faster and better performing than RNN architectures. And they're thus finding traction in the same set of domains. Sequence based problems such as natural language processing, genomics, and motion tracking.
